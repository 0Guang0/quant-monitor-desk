### P3 — LOW / 计划外 NON-BLOCKING

#### [P3] A6-NB-1 — Loader 证据文件无 row 硬顶

- **Area:** Loading / Network（本地 I/O）
- **Location:** `rehearsal_loader.py` — `_load_baostock_bundle` / `_load_fred_bundle`（`json.loads` 全量）
- **Description:** Plan/loader 限制 **symbol 数量**，但未对 `bars.json` / `observations` 数组长度设硬顶；`dry_run=false` 且传入巨大自定义 `evidence_dir` 时，内存与 CPU（`sorted(dates)`）随文件线性/超线性增长。
- **Impact:** potential impact — 恶意或误配证据目录可导致排练进程内存尖峰；**不影响**默认 fixture 路径（2 行 / 1 obs）。
- **Recommendation:** R3G-02 或 live 路径硬化时增加 `R3G01_MAX_EVIDENCE_ROWS`（例如 ≤1000，与 `market_bar_p0` 对齐）并在 loader 入口 reject；当前 sandbox audit 可接受。

#### [P3] A6-NB-2 — 生产 mutation proof 全库扫描成本

- **Area:** Network（本地 DuckDB I/O）
- **Location:** `rehearsal_runner.py:336-339` + `mutation_proof.build_production_mutation_proof`
- **Description:** 排练开始读 `key_table_row_counts`；结束 `build_production_mutation_proof` 再执行 `all_table_row_counts`（每表 COUNT）+ 两次 `read_bytes()` 全文件 hash。
- **Impact:** potential impact — 生产 DuckDB 越大，单次排练耗时/内存越高；与 `staged_pilot` 共享模式，非 R3G 独有。
- **Recommendation:** 维持现状（审计需强证明）；若未来频繁自动化排练，可复用 before 快照避免二次 `all_table_row_counts`（runner 已传 `before_counts`/`before_bytes`，但 after 仍全扫）。

#### [P3] A6-NB-3 — ResourceGuard PAUSE 不阻断排练

- **Area:** Rendering / JS（资源编排）
- **Location:** `rehearsal_runner.py:343-344`（仅 `HARD_STOP`）
- **Description:** `Decision.PAUSE` / `WARN` 时排练继续；与 `staged_pilot` route preview 一致，弱于 `DataSourceService.fetch` 对 PAUSE 的阻断。
- **Impact:** potential impact — 资源紧张时人工排练仍可能叠加 DuckDB 写与 DH profile；sandbox tmp DB，无生产写。
- **Recommendation:** 操作员手册注明：PAUSE 时宜手动推迟排练；R3G-03 生产入口须复评是否 PAUSE 阻断。

#### [P3] A6-NB-4 — `--allow-live-fetch` 无 fetch 实现

- **Area:** Network
- **Location:** `rehearsal_runner.py:328-329, 346`；`data_commands.sandbox_clean_write_rehearse`
- **Description:** CLI 暴露 `allow_live_fetch`；runner 仅切换 `staged_fixture_mode`，**未**调用 `DataSourceService.fetch` 或 FRED live port。
- **Impact:** potential impact **正面** — 当前无 live 网络风险；未来接线时必须挂载 RG + FRED `MAX_*` caps + per-series row limit。
- **Recommendation:** 实现 live fetch 前对照 `fred_sandbox_pilot` / `staged_pilot` live 路径与 contract caps；本切片 SKIP 不受影响。
