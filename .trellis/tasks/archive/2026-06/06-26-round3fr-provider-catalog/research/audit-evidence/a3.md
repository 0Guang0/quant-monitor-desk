## 3. `load_provider_catalog` 路径安全

**文件：** `backend/app/datasources/provider_catalog.py` L18–31

```python
resolved = yaml_path.resolve()
root = PROJECT_ROOT.resolve()
if not resolved.is_relative_to(root):
    raise ProviderCatalogError(...)
raw = yaml.safe_load(resolved.read_text(encoding="utf-8")) or {}
```

| 控制                               | 评估                                                 |
| ---------------------------------- | ---------------------------------------------------- |
| `resolve()` 先于边界比较           | ✓ 防 `../` 与符号链接逃逸                            |
| `is_relative_to(PROJECT_ROOT)`     | ✓ Python ≥3.11（`pyproject` 要求 3.11+）             |
| `yaml.safe_load`（非 `yaml.load`） | ✓ 防任意对象反序列化                                 |
| `path` 参数信任边界                | 仅测试/内部调用；diff 内无 CLI/API 暴露用户可控 path |

**调用面（worktree）：**

- `provider_for_source` → 默认 `load_provider_catalog()`（无外部 path）
- `tests/test_provider_catalog.py` 使用默认路径
