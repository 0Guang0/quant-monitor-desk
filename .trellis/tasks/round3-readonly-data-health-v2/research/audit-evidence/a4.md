## DOUBT（错误处理 / 边界）

1. **损坏 JSON 与缺失同形（DQ-1）：** `_read_profile_json` 捕获 `json.JSONDecodeError` 后返回 `None`，上层一律生成 `fred_evidence.json missing` 类 FAIL，运维无法区分「无文件」与「半写入损坏」。（`data_health.py:836-844` → `879-890`）
2. **BLOCKED 项 severity=WARN：** `_blocked_check` 设 `severity="WARN"`、`status="BLOCKED"`；按 severity 过滤的下游会低估 BLOCKED。（`data_health.py:187-203`）
3. **rollup 汇总项 severity 与 status 不对齐：** 子 profile `overall_status=FAIL` 时，`ROLLUP_*` 行 `severity="WARN"` 但 `status="FAIL"`。（`data_health.py:1126-1139`）
4. **v3 profile 未读 manifest 内 schema/conflict：** R3E §2 要求 v3 检查 schema/conflict summary，但 `_checks_staged_pilot_v3` 仅 caps/role/PDF；`compliant/pilot_v3_manifest.json` 的 `conflict_summary` 未被消费（rollup 层 `safety_evidence` 单独兜底）。（`data_health.py:1006-1070`；`staged_pilot_v3/compliant/pilot_v3_manifest.json:37`）
