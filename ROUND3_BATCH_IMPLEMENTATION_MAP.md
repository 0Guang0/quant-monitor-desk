# Round 3 Batch Implementation Map

> Purpose: one root-level index for splitting Round 3 into executable Trellis batches without losing task/deferred-item traceability.
>
> Status: planning/index document. It is not runtime code and does not participate in DB migration.
>
> Scope rule: `docs/` and `specs/` are design/contract inputs only. Runtime implementation must land in `backend/`, `frontend/`, `scripts/`, `configs/`, `tests/`, or other implementation paths already mapped by `MIGRATION_MAP.md`.

## 1. ID convention used by this file

Round 3 has two classes of work:

1. **Formal numbered implementation tasks:** `017`-`023` under `docs/implementation_tasks/ROUND_3_MODELING_LAYERS/`.
2. **Unnumbered or deferred repay items:** registry IDs from `docs/AUDIT_DEFERRED_REGISTRY.md` / `docs/UNRESOLVED_ISSUES_REGISTRY.md`, plus a small number of local index aliases for items that have no formal task number.

## 6. Trellis batching constraints

Each batch should become its own Trellis complex task unless a later human maintainer intentionally splits it further.

For every batch Trellis plan:

1. Plan must read this file, `MIGRATION_MAP.md`, `docs/implementation_tasks/TASK_INPUT_CONTEXT_INDEX.md`, the relevant original task files, the relevant registry rows, and the relevant design/contract/rule/definition docs from the Batch Plan Source Index above.
2. `MASTER.plan.md` must include a child deliverable table mapping each item ID above to an acceptance criterion.
3. `MASTER.plan.md` must include a Source Context Index that states which sources were summarized, which must be read in original form, which were filtered as stale/out-of-scope, and why.
4. `AUDIT.plan.md` must include an Audit Source Trace from item ID → source document → acceptance criterion → code/test evidence → registry update.
5. `implement.jsonl` first entry must be `MASTER.plan.md`; subsequent entries must be limited to files explicitly marked Execute must-read by MASTER. Do not default-include original task cards, root maps, registries, or docs/specs that MASTER already summarized losslessly.
6. `audit.jsonl` / `check.jsonl` must be limited to files explicitly marked Audit must-read by `AUDIT.plan.md` or MASTER Source Context Index.
7. Execute must not use docs/specs as implementation paths.
8. Any item not closed must remain visible as `DEFERRED` with a concrete phase, owner/task hook, and closure test.
