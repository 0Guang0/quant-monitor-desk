### OOB-1 — closeout `false` vs `null`（DB 缺失）

| 维度                 | 核查结果                                                                                                                                                                                                         |
| -------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **事实**             | production DB 缺失时，`mutation_proof.py:97-107` 返回 `db_hash_unchanged: None`、`row_counts_unchanged: None`、`proof_status: INCONCLUSIVE`                                                                      |
| **closeout 序列化**  | `build_pilot_v2_closeout`（`staged_pilot.py:1994-2006`）用 `hash_ok = … is True` / `counts_ok = … is True` 二值化 → JSON **`false`**（非 `null`）                                                                |
| **execute-evidence** | `pilot_v2_closeout.json` L3/L13：`false`；`no_mutation_proof_v2.md` L7-8：`None`；`validation_report_v2.json` L25/L29 内嵌 `production_mutation_proof`：`null`                                                   |
| **gate 行为**        | `closeout_pass = hash_ok ∧ counts_ok ∧ proof_status == "VERIFIED"` → DB 缺失时 **`closeout_pass: false`** + **`mutation_proof_status: INCONCLUSIVE`** — fail-closed 正确                                         |
| **单测覆盖**         | `test_stagedPilotV2_closeoutRequiresHashAndRowCountsUnchanged` 覆盖 VERIFIED+True / MUTATION_DETECTED+False；`test_stagedPilotV2_closeoutMatrix_allSourcesClassified` 传入 `None` 但未断言 closeout 输出 bool 值 |
| **MASTER §6.1**      | 字段类型声明为 `bool`；AUD-08 gate「PASS 须 `true`」— 未规定 absent-DB 时应保留 `null` 三态                                                                                                                      |

**A5 判定（OOB-1）：** **NON-BLOCKING · gate 功能与 AC-MUT-001 / AUD-08 一致 · 序列化语义跨产物不一致**

- **与 AUD-08 / AC-MUT-001：** ✅ **一致** — `false` 在 gate 语境下等价于「未证明通过」，与 `MUTATION_DETECTED` 路径同样阻断 `closeout_pass`；`mutation_proof_status: INCONCLUSIVE` 提供主状态，不构成假 PASS。
- **与 `no_mutation_proof_v2.md`：** ❌ **不一致** — proof 层用 `None`（未检测），closeout 层用 `false`（读者可能误读为「已检测且变异」）。
- **与 `validation_report_v2.json`：** ❌ **不一致** — 同一次 run 内嵌 proof 为 `null`，closeout 为 `false`。
- **是否应改为 `null`：** 从 **诚实三态** 角度，`null` 更准确区分 `INCONCLUSIVE`（未验证）与 `MUTATION_DETECTED`（已检测失败）；当前 MASTER §6.1 `bool` 类型与 `is True` 实现 **有意二值化**，属文档/契约缺口而非 gate 失效。
- **对 A5 追溯：** 不扣 AC 分；§3.5 AC-SP2-09 / AC-MUT-001 仍以 `closeout_pass: false` + 对抗测闭环为准。
