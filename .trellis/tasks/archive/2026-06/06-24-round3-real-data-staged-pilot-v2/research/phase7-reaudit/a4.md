1. **三态丢失导致同形异义：** DB 缺失（`None`→`false`+`INCONCLUSIVE`）与真实变异（`False`+`MUTATION_DETECTED`）在 closeout 的 `db_hash_unchanged`/`row_counts_unchanged` 字段上 **同为 JSON `false`**，仅靠 `mutation_proof_status` 区分。自动化消费者若只读 bool 字段会误判。（`staged_pilot.py:1994-2006`；`pilot_v2_closeout.json:3-5,13`）

## OOB 复核（用户指定 · OOB-1）

### 问题陈述

> DB 缺失时，`build_pilot_v2_closeout` 是否把 `db_hash_unchanged`/`row_counts_unchanged` 的 `None` 写成 `false`？与 `no_mutation_proof_v2.md` 的 `None`/INCONCLUSIVE 语义是否一致？

### 证据链（file:line）

| 阶段                         | 行为                                                                                                                        | 证据                                                       |
| ---------------------------- | --------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------- |
| 1. mutation proof（DB 缺失） | `proof_status="INCONCLUSIVE"`；`db_hash_unchanged=None`；`row_counts_unchanged=None`；`reason="production_db_file_missing"` | `mutation_proof.py:97-108`                                 |
| 2. markdown 证据             | 直写 `None` 字面量                                                                                                          | `no_mutation_proof_v2.md:6-8`；`staged_pilot.py:1924-1926` |
| 3. closeout 构建             | `hash_ok = ... is True` → `None` 为 **False**；`counts_ok` 同理；写入 payload                                               | `staged_pilot.py:1994-2006`                                |
| 4. closeout 落盘 JSON        | `"db_hash_unchanged": false`，`"row_counts_unchanged": false`，`"mutation_proof_status": "INCONCLUSIVE"`                    | `pilot_v2_closeout.json:2-5,13`                            |
| 5. gate 结果                 | `closeout_pass=false`（因 `proof_status != "VERIFIED"`，且 bool 亦为 false）                                                | `staged_pilot.py:1997`；`pilot_v2_closeout.json:2`         |

### 语义对照

| 工件                      | `db_hash_unchanged` | `row_counts_unchanged` | `proof_status` / `mutation_proof_status` | 读者含义                                       |
| ------------------------- | ------------------- | ---------------------- | ---------------------------------------- | ---------------------------------------------- |
| `mutation_proof` dict     | `None`              | `None`                 | `INCONCLUSIVE`                           | **未观测**（文件不存在）                       |
| `no_mutation_proof_v2.md` | `None`              | `None`                 | `INCONCLUSIVE`                           | 与 dict 一致                                   |
| `pilot_v2_closeout.json`  | `false`             | `false`                | `INCONCLUSIVE`                           | bool 像「已检测且未通过」；status 纠正为未证明 |

**结论（事实）：** 是，`build_pilot_v2_closeout` **确实** 将 `None` coerce 为 `false`（`is True` 模式）。与 `no_mutation_proof_v2.md` 的 `None` 语义 **不一致**——markdown 表达「未观测」，closeout JSON 表达「bool gate 未通过」。

**结论（安全）：** gate **fail-closed 正确**——无论 bool 如何写，`proof_status != "VERIFIED"` 已阻止 `closeout_pass`。不构成 production-live 误放行。

**结论（契约）：** MASTER §6.1 要求 bool 类型，实现 **符合计划 schema**；计划本身未要求 absent-DB 时保留 `null` 三态。
