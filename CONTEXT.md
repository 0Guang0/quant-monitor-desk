# Quant Monitor Desk

本地优先的量化监控台：登记数据源能力、按策略选源取数，并在故障时诚实降级与连续观察。

## Language

### 数据源启用

**开关本（activation overlay）**：
管理员留下的、可审计可撤销的「是否允许某源参与某领域/操作」记录；不是能力目录本身。
_Avoid_: 内存 override、force_enable、偷开开关、临时 patch

**有效启用**：
基础登记状态与开关本等规则合成后、路由真正可见的「现在能不能用」。
_Avoid_: 仅看 YAML `is_enabled`、仅看测试 monkeypatch

**安检（路由门控）**：
在开关本之后检查执照、平台、能力、资源护栏等；任一项不过即失败关闭。
_Avoid_: 与开关本混在一个「顺便全开」的函数里

**编排壳**：
某源取数流水线上的专用组装（如按系列灌水位线）；不是启用策略。
_Avoid_: 把 builder / proxy 当成另一套启用权威

### 运行档位（证据）

**产品默认**：
未写测试开关本、未隔离根时的真实默认策略结果。
_Avoid_: 用沙箱 READY 冒充产品已启用

**沙箱验收（gate_live）**：
在隔离数据根上、可含测试开关本的验收运行；不得升格为产品默认证明。
_Avoid_: live 字样但无隔离根、fixture 冒充发布
