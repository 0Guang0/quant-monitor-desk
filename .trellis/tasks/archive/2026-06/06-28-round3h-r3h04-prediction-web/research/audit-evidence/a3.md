### [LOW] Kalshi 可作 prediction 域 Primary route READY，缺 orchestrator 级 clean-write 负例

- **Location:** `tests/test_prediction_market_adapters.py` L185–204；`source_registry.yaml` `prediction_market_probability.primary: kalshi`（L658–661）
- **Description:** 活卡要求 kalshi「route 不得选 clean-table writer」；当前仅有 port bundle 无 `clean_write_target` 与 registry `mark_missing` fallback 测试，无 staged pilot / sync orchestrator 级负例证明 prediction 域 fetch 不会进入 `write_manager`。
- **Impact:** 新 port 本身不调用 `writer()`，且 `backend/app/sync/**` 无 `prediction_market`/`web_search` 引用；**当前无 silent clean 写路径**，但防御深度依赖 schema 边界而非端到端 route→write 测试。
- **Proof of concept:** 若未来 orchestrator 为 `prediction_market_probability` 接 generic clean promote，kalshi Primary READY 可能成为入口——今日代码未见该接线。
- **Recommendation:** A4/A8 或合并后补一条 orchestrator/service path 负例：kalshi/web_search fetch 后 `assert_can_write` 仍为 false。

### [LOW] Polymarket live 无 API key 仍发起外联（已 env 门禁）

- **Location:** `polymarket_port.py` L106–117
- **Description:** Live gamma API GET 无认证头；依赖 `POLYMARKET_LIVE_SMOKE` + 授权 YAML。
- **Impact:** 误设 env 时仍仅访问固定 `gamma-api.polymarket.com` + 白名单 slug；**无用户可控 URL SSRF**。
- **Recommendation:** 保持 mock-first；live 切片可加 rate-limit 与响应大小 cap（与 kalshi 对齐）。

### [INFO] domain_roles YAML 与 web_search validation_only 双轨

- **Location:** `source_registry.yaml` L727–735（`supplemental_web_evidence.primary: web_search` + 源级 `validation_only: true`）
- **Description:** YAML 列 primary 但 route planner L154–158 运行时 `validation_only_cannot_be_primary` 阻断。
- **Impact:** 运维读 YAML 可能误判；runtime fail-closed 已测（`test_web_search_port_route_validationOnlyPrimaryBlockedWhenEnabled`）。
- **Recommendation:** registry 协调切片统一 validation-only 源槽位（非本卡阻断）。
