# Global Resource Limits

本项目默认运行在用户本机，必须低占用、可暂停、可恢复，不能影响用户正常使用电脑。

## 1. 默认模式

默认模式必须为 `eco`。

| 模式   | 适用场景           |       CPU 线程 | 进程内存软上限 | DuckDB memory_limit | 临时目录上限 |
| ------ | ------------------ | -------------: | -------------: | ------------------: | -----------: |
| eco    | 默认、本机日常使用 |            1-2 |         约 1GB |         768MB-1.5GB |          2GB |
| normal | 用户主动盘后运行   |            2-4 |         约 2GB |             1.5-3GB |          5GB |
| batch  | 用户明确回补/重建  | 不超过半数 CPU |         约 4GB |               3-6GB |         10GB |

## 2. 硬暂停条件

以下情况必须暂停非核心任务：

- 系统可用内存 < 2GB。
- 磁盘剩余 < 20GB。
- 项目目录超过 25GB。
- CPU 长时间高占用影响用户体验。

以下情况只能允许安全退出、审计写入和用户确认：

- 系统可用内存 < 1GB。
- 磁盘剩余 < 10GB。
- 项目目录超过 40GB。

## 3. 禁止事项

- 禁止默认全市场全历史扫描。
- 禁止前端触发大范围 backfill。
- 禁止 Agent 触发大查询。
- 禁止将全历史数据读入 Pandas。
- 禁止无分页 API。
- 禁止无限保留 cache / temp / logs。

## 4. 资源保护输出

任务被资源保护暂停时，必须输出：

- `RESOURCE_GUARD_PAUSED`
- 触发原因。
- 当前可用内存 / 磁盘 / 项目目录大小。
- 建议用户稍后运行或切换 normal/batch 模式。
