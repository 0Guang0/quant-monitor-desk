## 2. SKIP 理由（§3.6 等价）

### 2.1 AUDIT.plan 冻结理由（复述 + 证实）

1. **无 hot path / SLA** — `MarketStructureBuilder` 仅被 `tests/test_layer4_market_structure.py` 调用；无 FastAPI 路由、无 `production_equivalent_smoke.py` 路径、无生产调度/CLI。
2. **staged fixture 微型数据面** — fixture 含 **1** 条 `calendar.json` 行、**1** 条 `breadth.json` 映射、**7** 行 `manifest.yaml`；单次 `build()` 产出 8 个 registry 种子行 + 日历 tuple + 1 breadth + lineage envelope（均为 dataclass）。
3. **零重 I/O 栈** — 无 DuckDB / Parquet / HTTP / live fetch；磁盘读为 **3×** 小文件（`manifest.yaml`、`calendar.json`、`breadth.json`），每次 `build()` 全文 `read_text` + parse。
4. **MASTER §10 无 perf 冻结行** — Tier A/B 为 `pytest` + `ruff` + `uv sync`；无时长/内存上限数字可供「指标 | 阈值 | 实测」表 PASS/FAIL。
5. **与 R3Y pilot caps 解耦** — R3Y `max_symbols` / `max_rows_per_source_domain` 约束 `staged_pilot.py` / vendor fetch；**022 diff 未触及** `ops/staged_pilot*`、`datasources/` live 路径（forbidden + A3 范畴）。

## 3. 计划外 perf 风险扫描

> 按 `audit-adversarial-authority.md` A6 最低对抗动作：**即使 SKIP，仍须扫描** hot path、无界 I/O、批大小/内存尖峰、全市场扫描面。

### 3.1 Hot path

| 路径                                  | 是否 hot   | 证据                                             | 风险                           |
| ------------------------------------- | ---------- | ------------------------------------------------ | ------------------------------ |
| `MarketStructureBuilder.build()`      | **否**     | 仅 `tests/test_layer4_market_structure.py`；7 测 | 当前无生产 SLA                 |
| `GET /api/layer4/*`                   | **未实现** | MASTER §3.2 defer Round 4+                       | 未来 API 须单独冻结 p95        |

**结论：** 本分支**不存在**可观测的生产 hot path；SKIP 合理。

### 3.2 无界 I/O / 全市场扫描

| 面                      | 扫描方法               | 发现                                                                                  | 评级                 |
| ----------------------- | ---------------------- | ------------------------------------------------------------------------------------- | -------------------- |
| 网络 / live fetch       | `rg` `layer4_markets/` | 无 `http`/`requests`/`DataSourceService`；`source_fetch_ids` 仅为 lineage 元数据      | **无**               |
| DuckDB / Parquet 全表扫 | `rg` + 模块审阅        | 零 DB/Parquet 引用                                                                    | **无**               |
| 全市场 adapter 拉取     | `_adapter_for_market`  | 仅 `CN_A` → `StagedCNAMarketAdapter`；其余 7 `market_id` 仅为 `REGISTRY_SEEDS` 元数据 | **无扫描面**         |
| 文件读取                | 代码审阅               | 每 `build()` 读 manifest + calendar 全列表 + breadth 单对象                           | **有界（3 小文件）** |
| calendar 行数上限       | 代码审阅               | `_load_calendar_rows` 无 `max_rows`；靠 `staged_fixture_only` 信任边界                | 见 NB-2              |
| 测试 `copytree`         | mutation 用例          | 仅 `.audit-sandbox/layer4_market_mutations/` 复制微型 fixture                         | **有界**             |

**结论：** **无** live 全市场扫描、**无** DuckDB 无界 I/O；AC-022-6 + manifest 双闸与 playbook §8.2「ResourceGuard / eco 默认未触发全市场扫描」一致。

### 3.3 批大小 / 内存尖峰

| 面                           | 复杂度 / 行为                 | fixture 量级            | 计划外风险                                       |
| ---------------------------- | ----------------------------- | ----------------------- | ------------------------------------------------ |
| registry 种子                | 固定 `REGISTRY_SEEDS` 8 行    | 8                       | O(1) 常量，非无界                                |
| calendar 加载                | O(n) 列表 + PK 去重           | n=1                     | 巨型 calendar.json → 内存尖峰；无 builder 侧硬顶 |
| breadth 加载                 | O(1) 单 dict                  | 1                       | 可忽略                                           |
| 输出物                       | dataclass tuple，无 DataFrame | 固定规模                | 无大副本                                         |
| manifest 解析                | 全文件 `yaml.safe_load`       | <0.5KB                  | 巨型 YAML 无 ResourceGuard 钩子                  |
| `collect_result_field_names` | O(rows×fields) 字段名扫描     | ponytail 注释已标天花板 | AC-022-7 边界测；非热路径                        |

**结论：** 当前 fixture 下内存/CPU **可忽略**（最慢 pytest call 0.01s）；结构性风险留待 Batch 6 CLI / 多市场 adapter 化时偿还，**非 022 BLOCKING**。
