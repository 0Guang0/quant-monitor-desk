## 2. SKIP 理由

### 2.1 AUDIT.plan 冻结理由（复述 + 证实）

1. **无 hot path / SLA** — v3 入口为 `staged_pilot.py` 内 `run_pilot_v3_*` / closeout CLI 路径；无调度器、无 API 路由、无用户-facing 延迟承诺。
2. **受控 caps** — `pilot_v3_caps.json` + 代码常量：`max_symbols_baostock=20`、`max_trade_days=120`、`max_network_calls_per_run=50`、`full_market_scan=false`（见 `execute-evidence/pilot_v3_caps.json` 与 `staged_pilot.py` v3 段 `MAX_*_V3`）。
3. **任务卡 scope 排除重 I/O** — `R3E_real_data_staged_pilot_v3.md` §1：非全市场接入、非 production clean write；akshare validation-only。
4. **MASTER §6 无 perf 冻结行** — 通过条件仅为 pytest/ruff/compileall exit 0；无时长/内存数字可供 A6 PASS/FAIL。
5. **v2 先例** — 同系列 staged pilot v2 已 A6 SKIP（`archive/.../06-24-round3-real-data-staged-pilot-v2/research/audit-evidence/a6.md`）；v3 延续 pilot 性质，仅扩 WL 驱动样本上限。

### 2.2 实现路径 vs 性能敏感面

| 组件                                        | v3 主路径是否调用                      | 数据量级                                         |
| ------------------------------------------- | -------------------------------------- | ------------------------------------------------ |
| `load_pilot_v3_whitelist_scope` / caps 校验 | **是**                                 | WL YAML 五文件；symbol 数 ≤ caps                 |
| `run_pilot_v3_baostock` / cninfo / akshare  | **是**（授权 live 或 mocked）          | ≤20 symbols × ≤120 trade days                    |
| `capture_raw_and_staging_evidence`          | **是**                                 | sandbox manifest；`production_clean_write=false` |
| `ResourceGuard`                             | **是**（v1/v2 路径；v3 单测多为 mock） | HARD_STOP 已存在；无 SLA 数字                    |
| DuckDB clean table 写入                     | **否**                                 | `no_mutation_proof_v3.md` 声称无 prod 写         |

**结论：** 当前分支为 **bounded sandbox evidence capture**，非生产热路径；与 AUDIT.plan「pilot 无 SLA」一致。

## 3. 计划外 perf 风险扫描

> 按 `audit-adversarial-authority.md`：**即使 SKIP，仍须扫描** hot path、无界 I/O、批大小/内存尖峰。

### 3.1 Hot path

| 路径                          | 是否 hot   | 证据                                             | 风险            |
| ----------------------------- | ---------- | ------------------------------------------------ | --------------- |
| `run_pilot_v3_*` / closeout   | **否**     | 手动/CI 触发；非默认 daemon                      | 无生产 SLA      |
| baostock/cninfo live fetch    | **条件性** | 须 `live_authorization_*.yaml`；caps 50 网络调用 | 授权外为 mocked |
| production ingestion pipeline | **未触及** | `production_clean_write=false`                   | 正交            |

**结论：** 本分支**不存在**可观测的生产 hot path；SKIP 合理。

### 3.2 无界 I/O

| 面            | 扫描                | 发现                                          | 评级     |
| ------------- | ------------------- | --------------------------------------------- | -------- |
| 全市场扫描    | caps + 代码         | `full_market_scan=false`；WL 驱动 symbol 列表 | **无**   |
| PDF 批量下载  | 单测 + 任务卡       | `test_v3_cninfo_rejects_pdf_expansion`        | **无**   |
| 网络调用      | `NetworkCallBudget` | `max_network_calls_per_run=50` 硬顶           | **有界** |
| manifest JSON | evidence 写盘       | 受 row/symbol caps 约束                       | 见 NB-1  |

### 3.3 批大小 / 内存尖峰

| 面                 | 行为             | 当前量级                           | 计划外风险              |
| ------------------ | ---------------- | ---------------------------------- | ----------------------- |
| baostock 日线拉取  | per-symbol fetch | ≤2000 rows/source                  | 未来扩 cap 须主会话排队 |
| cninfo metadata    | issuer 列表      | ≤500 rows                          | 同上                    |
| akshare validation | 对照样本         | ≤1000 rows；非 Primary             | 已 taxonomy 约束        |
| 并发多 agent fetch | playbook §2.5    | 单 worktree 独占 `staged_pilot.py` | 无跨分支争用            |

**结论：** caps 在代码与 JSON 双重体现；当前规模下 CPU/内存可忽略。
