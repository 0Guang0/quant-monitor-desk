## Findings

### [P2] registry 血缘检查 fail-open（未知 source 静默跳过）

- **Location:** `backend/app/ops/data_health.py:453-469`
- **Description:** `check_lineage_entry` 在 `reg.get(resolved_source)` / `reg.get(primary)` 外包 `except Exception: pass`，`SourceNotFoundError` 或 registry 加载异常时 **不** 产生 `DISABLED_SOURCE_USED` / `VALIDATION_ONLY_AS_PRIMARY` FAIL。
- **Impact:** 对抗性 evidence manifest 可使用 registry 外 `source_id`，绕过 disabled / validation-only 门禁；与 R3Y Red Flag「启用 disabled source」的检测意图部分失效。
- **Proof of concept:** manifest 中 `source_used: "totally_fake_source"` 且补齐 fetch_id/hash/as_of → lineage 字段 PASS，registry 规则被跳过。
- **Recommendation:** 对 `SourceNotFoundError` fail-closed（例如 `UNKNOWN_SOURCE` FAIL）；仅对明确预期的 registry 错误降级，或 `except SourceNotFoundError` 单独 FAIL。
- **Blocking:** **NON-BLOCKING**（staged-only v1；无写库/fetch 链；测试已覆盖已知 disabled/validation-only 源）

### [P2] evidence 路径未 sandbox 校验（R3Y-DH-02 缺口）

- **Location:** `backend/app/ops/data_health_cli.py:15-31` · `data_health.py:196-205`
- **Description:** `--evidence` 接受任意 `Path`，无 `resolve().is_relative_to(PROJECT_ROOT)` 或 `QMD_DATA_ROOT` 约束；`db_inspector._count_files_under` 已有 sandbox 先例（`db_inspector.py:267-274`）。
- **Impact:** 本地 ops 操作者可指向项目外目录；若该目录含合规 manifest 文件名，则只读解析其 JSON（信息泄露面限于 manifest 结构，非任意文件读）。
- **Recommendation:** v1.1 增加与 R3Y-DH-02 一致的 project-relative 校验；越界返回 exit 2 + 明确 error。
- **Blocking:** **NON-BLOCKING**（本地可信操作面；C-20 无 network 暴露）

### [P3] CLI `--output` 任意路径写

- **Location:** `backend/app/ops/data_health_cli.py:38-45`
- **Description:** 报告可写入操作者指定任意路径（仅创建/覆盖报告内容，非 DB）。
- **Impact:** 误操作覆盖；属 ops CLI 常规能力。
- **Recommendation:** 文档化；可选限制为 cwd 子路径或需 `--force`。
- **Blocking:** **NON-BLOCKING**

### [P3] manifest JSON 无读大小上限

- **Location:** `backend/app/ops/data_health.py:192-193` (`path.read_text` + `json.loads`)
- **Impact:** 恶意超大 manifest 可能导致内存压力（本地 DoS）。
- **Recommendation:** 复用项目 ResourceGuard / 读前 `stat().st_size` cap（与 GLOBAL_RESOURCE_LIMITS 对齐）。
- **Blocking:** **NON-BLOCKING**
