## 2. SKIP 理由（§3.6 等价）

### 2.1 AUDIT.plan 冻结理由（复述 + 证实）

1. **无 hot path / SLA** — `IndustryChainSnapshotBuilder` 仅被单元测试与未来 Batch 6 CLI 骨架调用；无 FastAPI 路由、无 `production_equivalent_smoke.py` 路径、无生产调度。
2. **staged fixture 微型数据面** — loader fixture **2 anchors**（MSFT + OPENAI event_only）；L5 manifest **1 bar/anchor**；单次 `build()` 内存对象数个 dataclass。
3. **零重 I/O 栈** — 无 DuckDB / Parquet / HTTP / live fetch；唯一磁盘读为 **一个** `manifest.yaml`（`read_text` + `yaml.safe_load`）。
4. **MASTER §10 无 perf 冻结行** — Tier A 为 `pytest` + `ruff` + `compileall`；无时长/内存上限数字可供 A6 填「指标 | 阈值 | 实测」表。
5. **与 R3Y wt-r3-pilot-v2 解耦** — R3Y 任务卡 caps（`max_symbols: 3-5`、`max_rows_per_source_domain: 500`、`max_network_calls_per_run: 25`）约束 `staged_pilot.py` / vendor fetch；**021 diff 未触及** `backend/app/ops/staged_pilot*`、`datasources/` live 路径（A3/A7 已证）。

## 3. 计划外 perf 风险扫描

> 按 `audit-adversarial-authority.md` A6 最低对抗动作：**即使 SKIP，仍须扫描** hot path、无界 I/O、批大小/内存尖峰。

### 3.1 Hot path

| 路径                                   | 是否 hot   | 证据                                                  | 风险                    |
| -------------------------------------- | ---------- | ----------------------------------------------------- | ----------------------- |
| `IndustryChainSnapshotBuilder.build()` | **否**     | 仅 `tests/test_layer3_snapshot_builder.py` 调用；9 测 | 当前无生产 SLA          |
| `GET /api/layer3/*`                    | **未实现** | MASTER §3.2 defer Round 4+                            | 未来 API 须单独冻结 p95 |
| `layer3 build-snapshot` CLI            | **未实现** | MASTER §3.2 defer Batch 6                             | 未来须挂 smoke          |
| R3Y staged pilot fetch                 | **未触及** | diff 无 `staged_pilot.py`                             | R3Y caps 不适用本 diff  |

**结论：** 本分支**不存在**可观测的生产 hot path；SKIP 合理。

### 3.2 无界 I/O

| 面                      | 扫描方法                              | 发现                                                             | 评级               |
| ----------------------- | ------------------------------------- | ---------------------------------------------------------------- | ------------------ |
| 网络 / live fetch       | `rg` snapshot_builder + layer3_chains | 无 `http`/`fetch`/`duckdb` import                                | **无**             |
| DuckDB / Parquet 全表扫 | 同上 + A7                             | 零 DB 引用                                                       | **无**             |
| 文件读取                | 代码审阅                              | 每 `build()` 读 **1×** `manifest.yaml` 全文入内存                | **有界（单文件）** |
| manifest 体积上限       | 代码审阅                              | **无** `max_bytes` / 行数 cap；靠 `staged_fixture_only` 信任边界 | 见 NB-2            |
| loader 重复 I/O         | 调用链                                | `build()` 接收 `IndustryChainLoadResult`；**不**二次 load bundle | **无重复**         |
| 测试 `copytree`         | `test_layer3_snapshot_builder.py`     | 仅变异 manifest 用例用 `tmp_path` 复制微型 fixture               | **有界**           |

**结论：** 无网络/DB 无界 I/O；唯一 I/O 为小型 YAML，**在 staged 信任模型下可接受**。

### 3.3 批大小 / 内存尖峰

| 面                           | 复杂度 / 行为                                  | fixture 量级 | 计划外风险                                                                          |
| ---------------------------- | ---------------------------------------------- | ------------ | ----------------------------------------------------------------------------------- |
| anchor 循环                  | `O(anchors)`                                   | 2            | 未来全链锚点可达数十～数百；仍远小于 R3Y 500 行 cap 量级                            |
| bar 查找                     | `_bar_for_trade_date` **线性扫描** `bars` 列表 | 1 bar/anchor | 若违反「不保存全量历史」塞入长 `bars` 列表 → **O(bars×anchors)**；无 builder 侧硬顶 |
| 输出物                       | 三份 `tuple`（snapshots / lineage / mapping）  | 各 ≤ anchors | 无 DataFrame 大副本                                                                 |
| manifest 解析                | 全文件 `safe_load` 一次                        | <1KB         | 巨型 YAML 可导致内存尖峰；**无 ResourceGuard 钩子**                                 |
| lineage `parameter_hash_for` | 每 anchor 一次哈希                             | 2 次         | 可忽略                                                                              |

**结论：** 当前 fixture 下内存/CPU **可忽略**；**结构性**风险在于未来生产 manifest 未限时 `_bar_for_trade_date` 线性扫与无 YAML 体积 cap——属 **defer 到 Batch 5+/CLI** 的增强项，**非 021 BLOCKING**。

### 3.4 R3Y 任务卡交叉（wt-r3-pilot-v2）

| R3Y 要求                            | 与 021 关系                              |
| ----------------------------------- | ---------------------------------------- |
| `max_rows_per_source_domain: 500`   | 约束 pilot ingest；021 不读 pilot row 流 |
| `full_market_scan: forbidden`       | 与设计一致；021 staged-only 更强         |
| `production_clean_write: false`     | 021 无 WriteManager（A7 PASS）           |
| ResourceGuard caps 扩大样本前须批准 | 021 未扩大样本；仅测试 fixture           |

**结论：** R3Y caps **不覆盖** Layer3 snapshot builder；无冲突，无额外 A6 必跑项。
