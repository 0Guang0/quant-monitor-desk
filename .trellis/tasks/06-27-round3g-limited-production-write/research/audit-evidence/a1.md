### P1（AC/契约违反、缺失 fail-closed 门）

#### A1-P1-01 — CLI rollback 路径未与 approval `rollback_plan_path` 对齐

- **标题：** 四门锁缺第五对齐：`--rollback-plan` ≠ approval YAML 内 `rollback_plan_path` 仍可 promote
- **证据：** `approval_contract.py:65,83` 解析 `rollback_plan_path` 但 `validate_approval_contract`（L217-269）与 `run_limited_production_entry`（L401）均未将 CLI 传入路径与 contract 字段比对；活卡 §5 要求 approval 与 audit 在 `rollback plan path` 上完全一致
- **建议修复：** 在 `validate_approval_contract` 或 `run_limited_production_entry` 增加 `_resolve_path(contract.rollback_plan_path) == _resolve_path(request.rollback_plan)` 校验；补对抗测 `test_ApprovalContract_rollbackPlanPathMismatch_blocks`

#### A1-P1-02 — CLI audit 路径未与 approval `audit_decision_file` 对齐

- **标题：** `--audit-decision` 可与 approval 内声明的 `audit_decision_file` 不一致
- **证据：** `approval_contract.py:62,80` 解析 `audit_decision_file`；`validate_approval_contract`（L223-224）仅读取 CLI 传入路径，未校验与 YAML 字段一致；活卡 §5 要求 approval file 与 audit decision 同意 exact paths
- **建议修复：** 校验 `_resolve_path(contract.audit_decision_file) == _resolve_path(audit_decision_path)`；补对抗测

### P2（质量 / spec 漂移 / 弱覆盖）

#### A1-P2-01 — FRED `live_fetch` / `fred_authorization` 未接入 runner

- **标题：** `PromoteRequest.allow_live_fetch` / `fred_authorization` 声明但未在 `run_limited_production_entry` 使用
- **证据：** `limited_production_entry.py:80-81` 字段存在；`run_limited_production_entry`（L422-426）调用 `load_rehearsal_bundle` 未传 live-fetch 参数；对比 `rehearsal_runner.py:400-439` 有完整 FRED 授权链；活卡 §9.4 要求 live fetch 双条件
- **建议修复：** 将 `allow_live_fetch`/`fred_authorization` 传入 bundle 加载或显式 fail-closed 拒绝 FRED live 直至授权 artifact 校验通过

#### A1-P2-02 — 活卡 §10.1 `rehearse/audit prod-path regression` 未在 R3G-03 测试模块点名

- **标题：** §8.1 停止条件 #3 无 R3G-03 模块内回归测或 manifest 链接
- **证据：** `test_round3g_limited_production_clean_write.py` 无 `rehears`/`R3G-01` 引用；回归测存在于 `test_round3g_sandbox_clean_write_rehearsal.py:820-837`（R3G-01 自有模块）；全库 pytest 绿但未满足活卡 §9.7「必须覆盖」的模块级追溯
- **建议修复：** 在 R3G-03 测试加 parametrized import 或 §2.1 Tier A 命令文档化显式包含 R3G-01/02 rehearse/audit 子集

#### A1-P2-03 — `no_cap_expansion=false` 无专门对抗测

- **标题：** `block_if cap_expansion` 仅测 symbol 超限，未测 approval 标志位
- **证据：** `approval_contract.py:239-240` 检查 `no_cap_expansion`；测试仅有 `test_ApprovalContract_capExpansion_blocks`（symbol 数）与 agent 标记；无 `no_cap_expansion=false` fixture 测
- **建议修复：** 仿 `test_ApprovalContract_agentTriggeredWrite_blocks` 增 YAML 篡改测

#### A1-P2-04 — PromoteRunner WriteManager bypass 覆盖偏薄

- **标题：** §10.1「WriteManager/DbValidationGate bypass」仅测 `WriteManager(gate=None)` 构造，未测 promote 路径合成绕过
- **证据：** `test_PromoteRunner_writeManagerBypassRequiresExplicitGate`（L351-361）仅实例化测试；无 promote runner 内 gate skip 对抗 fixture
- **建议修复：** 参考 `test_round3g_pre_production_adversarial_audit.py:889` gate_bypass 模式增 promote 路径测

#### A1-P2-05 — 多源 pilot（1–3 sources）未实现

- **标题：** 活卡 §1 允许 1–3 sources，实现强制单 candidate
- **证据：** `approval_contract.py:250-252` `exactly one source candidate per promote run` + ponytail 注释；`load_approval_contract` 允许 ≤3 candidates（L161-162）但 validate 拒绝 >1
