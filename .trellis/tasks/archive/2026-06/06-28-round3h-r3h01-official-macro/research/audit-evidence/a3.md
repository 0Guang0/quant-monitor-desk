## Findings（按严重度）

### [INFO] GitNexus taint 层不可用

- **Location:** GitNexus `explain` / `pdg_query`
- **Description:** 仓库未记录 PDG/taint findings，`explain({target: fetch_ports})` 返回空。
- **Impact:** 自动化污点链审计缺失；需依赖静态审阅。
- **Recommendation:** 合并前运行 `node .gitnexus/run.cjs analyze --pdg`；对 `FredLiveFetchPort._live_observations` 重跑 `explain`。

### [LOW] FRED API key 经 URL query 传递

- **Location:** `backend/app/datasources/fetch_ports/fred_port.py` L131–141
- **Description:** `api_key` 作为 GET query 参数拼入官方 URL（FRED API 惯例）。
- **Impact:** 反向代理/访问日志可能记录 key；错误路径若记录完整 URL 有泄露风险（当前 `PortError` 不拼接 URL）。
- **Proof of concept:** 在含 query 日志的 HTTP 中间件后发起 live fetch → 日志可见 `api_key=`。
- **Recommendation:** live 扩面时：确保 fetch 日志 redact `api_key`；评估 FRED 官方是否支持 header 传递；延续 `test_data_adapter_contract` 类 redaction 测试覆盖 macro port。

### [LOW] SEC 身份头负例测试不完整

- **Location:** `tests/test_sec_edgar_adapter.py` L108–130
- **Description:** 仅测 env 缺失；未测「有值但缺联系信息」的负例（如 `QuantMonitor/1.0`）。
- **Impact:** 回归可能削弱 `_sec_user_agent` 启发式；**当前实现仍 fail-closed**（审计运行时验证）。
- **Recommendation:** 增补 `monkeypatch.setenv("SEC_EDGAR_USER_AGENT", "BareProduct/1.0")` 负例测试（无需改生产代码）。

### [LOW] `_resolve_raw_path` 未 jail 到 PROJECT_ROOT

- **Location:** `backend/app/ops/sandbox_clean_write/live_evidence_bridge.py` L43–49
- **Description:** 绝对路径存在且为文件时直接 `resolve()`，未校验是否在项目根下。
- **Impact:** 若攻击者能控制 staged manifest 中的 `raw_file_paths`，可读任意本地文件（需已有 manifest 写权限）。
- **Recommendation:** promote 扩面时用 `resolve_sandbox_path` 模式或 `is_relative_to(PROJECT_ROOT)` 约束；当前 pilot 信任边界内可接受。
