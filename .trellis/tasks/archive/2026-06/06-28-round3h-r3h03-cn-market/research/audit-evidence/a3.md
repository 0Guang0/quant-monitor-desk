### Findings

#### [MEDIUM] license_gate 仅校验环境变量存在，非真实授权证明

- **Location:** `backend/app/datasources/auth/license_gate.py:15-33`
- **Description:** `check_license_gate` 对 `qmt_xtdata` / `ths_ifind` / `qmt_xqshare` 仅检查 env 是否存在（如 `QMT_XTDATA_AUTHORIZED`、`THS_IFIND_LICENSE_ARTIFACT`），不验证 artifact 签名、终端会话或 vendor license 有效性。代码内 `ponytail:` 注释已声明此为已知上限。
- **Impact:** 拥有进程环境写权限的本地操作者可设置任意非空 env 绕过 gate，在 replay/mock 阶段拉取“授权正例” fixture；若未来切换 live vendor API 而未升级 gate，存在**误用未授权商业终端**的合规风险。
- **Proof of concept:**
  ```powershell
  $env:QMT_XTDATA_AUTHORIZED="1"
  # create_qmt_xtdata_fetch_port().fetch_payload(...) → 通过 gate（mock 路径）
  ```
- **Recommendation:** 保持 R3H-03 replay-first 现状可接受；live 启用前在 ADR 中落地 signed artifact / 终端握手校验，并将 `require_license_gate` 与 `platform_source_matrix` 的 `requires_user_confirmation` 对齐为同一授权 SSOT。

#### [LOW] source_registry.yaml 仍将 akshare 标为 3 个域的 yaml primary

- **Location:** `specs/datasource_registry/source_registry.yaml:431,469,477`（`macro_supplementary` / `cn_index` / `sector_board`）
- **Description:** 活卡 §2.8 #6 要求 akshare **永久 validation_only**；registry 源行 `validation_only: true` 已满足，但 `domain_roles` 仍写 `primary: akshare`。
- **Impact:** 当前 `SourceRoutePlanner.plan` 在 `role == "Primary"` 时强制 `validation_only_cannot_be_primary`（`route_planner.py:154-160`），运行时 fail-closed；但若未来维护者移除 `validation_only` 标志，yaml primary 绑定可导致 **silent primary 升格**。
- **Proof of concept:** 移除 `akshare.validation_only` 后，`cn_index` 域路由将可选中 akshare 为 Primary（无现有测试覆盖该回归）。
- **Recommendation:** 将三域 yaml primary 改为显式 validation 角色或文档化 `primary_candidate_blocked`；增加 parametrized 测试覆盖 `cn_index` / `sector_board`（已有 `macro_supplementary` 见 `test_r3x_residual_open_items_closure.py`）。

#### [LOW] ths_ifind 未列入 platform_source_matrix.yaml

- **Location:** `specs/contracts/platform_source_matrix.yaml`（无 `ths_ifind` 条目）
- **Description:** `SourceRoutePlanner._platform_allows` 对缺失条目返回 `platform_matrix_missing:{platform}:ths_ifind`（fail-closed），授权仅依赖 registry `enabled_by_default: false` + port `license_gate`。
- **Impact:** 无直接可利用漏洞；但平台矩阵与 QMT/xqshare 不对称，审计/trace 可读性下降。
- **Recommendation:** 为 `ths_ifind` 增加与 `qmt_xtdata` 同形的 matrix 行（`default_enabled: false`, `requires_user_confirmation: true`）。

#### [INFO] READY_WITH_EVIDENCE ≠ 无授权 route READY

- **Location:** `specs/datasource_registry/source_capabilities.yaml` — `qmt_xtdata` / `ths_ifind` / `qmt_xqshare` 均为 `READY_WITH_EVIDENCE`
- **Description:** 该状态表示 **port + replay fixture + 证据闭环** 已实现，不表示未授权用户可获得 `SourceRoutePlan.route_status == "READY"`。
- **验证:** 未授权时 registry `enabled_by_default: false`、platform gate、`license_gate` 三层阻断；pytest 负例全绿（见下）。

#### [INFO] qmt_xqshare 路由层未检查 QMT_XQSHARE_AUTHORIZED

- **Location:** `route_planner.py:72-77` vs `license_gate.py:19`
- **Description:** platform matrix 仅要求 `XQSHARE_REMOTE_HOST/PORT`；`QMT_XQSHARE_AUTHORIZED` 仅在 port fetch 时检查。
- **Impact:** 极端情况下（源被显式 enable + HOST/PORT 已设）路由或显示 READY，但 fetch 仍 `USER_AUTH_REQUIRED` — **fetch 边界 fail-closed**。
- **Recommendation:** 将 `QMT_XQSHARE_AUTHORIZED` 并入 matrix `requires_env` 或 `_platform_allows` 统一校验。
