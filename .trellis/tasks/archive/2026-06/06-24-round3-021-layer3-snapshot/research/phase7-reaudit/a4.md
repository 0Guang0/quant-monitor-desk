## DOUBT（对抗性边界审查）

**CLAIM：** as_of / event_only / lineage 三链在 staged fixture 路径下闭合，且无 P0 逻辑错误。  
**搜索范围：** `snapshot_builder.py` 全文、`lineage.py`、`models.py`、contract constraints、设计 §8.12.5–8.12.6、L2 `snapshot_builder.py` 对照、MASTER §5.3 未列边界。

### D-1 — event_only lineage 溯源指纹与 dataset_id 语义错位

**位置：** `snapshot_builder.py:57,74-76`

**问题：** `event_only` 的 `source_dataset_ids` 为 `staged:layer3_anchor:{anchor_id}`，但 `source_fetch_ids` / `source_content_hashes` 仍取自 **L5 manifest 全局** provenance（当前 fixture 仅为 MSFT bar）。OPENAI 未消费任何 L5 bar，lineage 却携带 MSFT 的 fetch/hash。

**影响：** 不违反 `LINEAGE_REQUIRED_FIELDS` 非空约束，但削弱「溯源指纹 ↔ 实际输入」一致性。

### D-2 — bar 字段缺失时 KeyError 泄漏

**位置：** `snapshot_builder.py:87-88,198-199`

**问题：** `bar["as_of_timestamp"]`、`bar["close"]` 直接下标；fixture 外畸形 YAML → `KeyError` 而非 `Layer3SnapshotError`。

### D-3 — `_parse_ts` 未校验 tz-aware

**位置：** `snapshot_builder.py:189-195`

**问题：** naive `datetime` 与 aware `as_of` 比较 → Python 3.12 `TypeError`，未被 `_reject_future_bar` 包装为 `Layer3SnapshotError`。

### D-4 — `float(bar["close"])` 异常类型不一致

**位置：** `snapshot_builder.py:88`

**问题：** 非数值 `close` → `ValueError`，与 manifest 其它路径的 `Layer3SnapshotError` 不一致，ops 侧难以统一捕获。

### DOUBT 结论

未发现 as_of / event_only / lineage **链断裂**或 P0 逻辑错误。D-1～D-4 均为 staged 子集可接受边界，**不构成 merge 阻断**。
