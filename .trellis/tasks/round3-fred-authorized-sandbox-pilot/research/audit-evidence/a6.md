## 2. SKIP 理由（§3.6 等价）

### 2.1 AUDIT.plan 冻结理由（复述 + 证实）

1. **受控小样本 sandbox pilot** — 最多 5 个 P0 series、每 series ≤100 行、每 run ≤10 次网络调用；无全市场/全 catalog 扫表。
2. **无生产 hot path / SLA** — `fred_sandbox_pilot.py` 仅被 pytest 与 execute-evidence 脚本调用；不经 FastAPI、不经 `SyncOrchestrator` 调度、不经 `production_equivalent_smoke.py`。
3. **零 DuckDB/Parquet 栈** — FRED pilot 写 sandbox JSON manifest；`fred_fetch_ports.py` 仅 urllib + `json.loads`；无 DB 连接、无分区扫描。
4. **任务卡 scope 排除重 I/O** — R3E §8：禁止全历史回填、禁止数千 series 自动发现、禁止定时轮询。
5. **MASTER §6 Tier A 无 perf 冻结行** — 通过条件 = pytest/ruff exit 0；无时长/内存数字可供 A6 PASS/FAIL。

**结论：** 本分支**不存在**可观测的生产 perf hot path；SKIP 合理。

### 3.2 无界 I/O

| 面 | 扫描 | 发现 | 评级 |
| --- | --- | --- | --- |
| 网络 / live fetch | `FredLiveFetchPort` | 每 series 单次 GET；`timeout=30`；`limit` 参数硬顶；默认 skip live | **有界** |
| DuckDB / Parquet | `rg` fred ops | 零引用 | **无** |
| JSON manifest 写 | `run_mock_fetch` | 全 manifest `json.dumps` 入内存；理论 ≤500 行 | **有界**（caps） |
| 全 FRED catalog | 代码审阅 | 无 auto-discovery；白名单 5 series | **无** |
| `response.read()` 全量读 | live port | API `limit` 约束响应体；100 行 JSON ≪ MB | 见 NB-3 |

### 3.3 批大小 / 内存尖峰

| 面 | 行为 | 当前量级 | 计划外风险 |
| --- | --- | --- | --- |
| 多 series 串行 fetch | `for series_id in request.series_ids` | ≤5 次迭代 | 无并行风暴 |
| manifest 累积 | `series_manifest` list append | ≤5 entries | 可忽略 |
| route preview registry load | 每次 `SourceRegistry.load()` + `CapabilityRegistry.load()` | 小 YAML | 见 NB-1 |
| live urllib body | `response.read()` 后 `json.loads` | ≤100 obs/series | 巨型 API 误配时尖峰；`limit` 参数缓解 |

**结论：** 声明 caps 下 CPU/内存**可忽略**；结构性风险在 Batch 6 接 `DataSourceService` 与缺 RG 挂载。
