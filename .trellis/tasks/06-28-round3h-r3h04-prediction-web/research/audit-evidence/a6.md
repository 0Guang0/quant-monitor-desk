## 1. SKIP 理由（AUDIT.plan §1 复述 + 证实）

### 1.1 Plan 冻结理由

1. **无 hot path SLA** — `AUDIT.plan.md` §1 将 A6 标为「不用 / SKIP」，说明为「无 hot path SLA；on-demand evidence API」。
2. **on-demand evidence API，非用户面 Web** — 三源实现为 `fetch_port.fetch_payload()` 按需拉取/回放证据 bundle；不经浏览器渲染、无 SPA 路由、无 CWV 观测面。
3. **mock/replay-first** — kalshi/polymarket/web_search 默认 mock；live 仅 `prediction_market_live_smoke.py` 双 env gate（`KALSHI_LIVE_SMOKE` / `POLYMARKET_LIVE_SMOKE` + authorization YAML）；web_search **无 live 路径**。
4. **硬 cap 有界** — frozen §7：markets ≤5 / window ≤30d / queries ≤3 / results ≤10；端口内 `reject_over_cap` + 白名单，禁止全市场/全 catalog 扫描。
5. **禁止主库写** — frozen §2.8：证据落 sandbox/replay；`manual_review_staging` 显式 `clean_write_permitted: False`。
6. **无冻结 perf 阈值** — Execute DoD = pytest + loop_maintain；无 smoke 时长/内存/LCP 数字可供 PASS/FAIL。

## 2. 轻量 on-demand API 扫描（SKIP 仍须执行）

> 按 Trellis A6 最低动作：即使 SKIP，仍扫 hot path、无界 I/O、O(n²) 等结构性反模式。

### 2.1 Hot path

| 路径 | 是否 hot | 证据 | 风险 |
| --- | --- | --- | --- |
| `Kalshi*/Polymarket*/WebSearch*MockFetchPort.fetch_payload` | **否** | pytest + replay fixture 主路径 | 无生产 SLA |

**结论：** 不存在可观测的生产 perf hot path；SKIP 合理。

### 2.2 算法复杂度 / O(n²)

| 位置 | 模式 | 上界 n | 复杂度 | 评级 |
| --- | --- | --- | --- | --- |
| 三 fetch port | 无嵌套 `for` | — | O(1) 每请求 | **无 O(n²)** |
| `reject_forbidden_resolution_fields` | `for key in payload` + `for signal in signals` × `for key in signal` | signals ≤5；键集固定小 | O(n·k)，k 常数 | **有界，非 O(n²)** |
| `build_bounded_widget_artifact` | 两次 list comprehension over `results` | ≤10 | O(n) | **有界** |
| `build_probability_signal_evidence_bundle` | `[_normalize_signal(row) for row in signals]` | ≤5 | O(n) | **有界** |
| `probability_signal.read_*` | 单次 `json.loads` + 线性 normalize | 文件体量由 cap 约束 | O(n) | **有界** |

**结论：** 未发现无界 O(n²) 或全表扫描；所有迭代均在 frozen cap 内。

## 3. Web Performance Audit 专向结论

本任务 **无浏览器 Web 应用面**（无 LCP 元素、无 JS bundle、无 layout shift 来源）。A6 在本轨语义为 **on-demand Python evidence fetch API** 的资源有界性审阅，而非 Core Web Vitals 实测。

### 3.1 Findings（静态 · potential impact）

#### [INFO] 无 Web 前端 — CWV 不适用
- **Area:** Core Web Vitals
- **Location:** 任务 scope 全体
- **Description:** R3H-04 交付 L3 fetch port + evidence normalizer；无用户可见页面。
- **Impact:** potential impact — N/A for LCP/INP/CLS
- **Recommendation:** 若未来挂载 dashboard widget，再对 bounded artifact 渲染做独立 A6。

#### [LOW] live port 先 `response.read()` 再解析
- **Area:** Network / Loading
- **Location:** `kalshi_port.py:133-134`, `polymarket_port.py:114-115`
- **Description:** urllib 全量读 body 后 `json.loads`；API 侧有单 market / `limit=1` 约束。
- **Impact:** potential impact — 恶意超大响应可导致内存尖峰；默认 mock 路径不受影响
- **Recommendation:** Batch 6 live primary 硬化时可加 `Content-Length` 上限或流式截断（**非本切片 BLOCKING**）。

#### [LOW] `build_bounded_widget_artifact` 对 results 双遍扫描
- **Area:** Rendering / JavaScript（等价：数据整形）
- **Location:** `manual_review_staging.py:40-51`
- **Description:** 分别构建 `rows` 与 `items` 两个列表。
- **Impact:** potential impact — n≤10 时可忽略；非 O(n²)
- **Recommendation:** 维持现状；若 results cap 将来放大再合并为单遍。

### 3.2 Positive Observations
- 全路径 **mock-first**，live 双 gate fail-closed。
- 端口级 **reject_over_cap** 与 frozen caps 一致；未改共享 `resource_guard.py`（BRANCH 边界）。
- 每请求 **单 instrument / 单 query**，无多市场串行风暴。
- **白名单** 阻断未知 market/query，避免无界 catalog 遍历。
- web_search 强制 `need_human_review=true` + `manual_review_state=queued`，无 clean 表热写。
