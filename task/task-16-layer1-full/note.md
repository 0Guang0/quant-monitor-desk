## 与 `findings.md` 的分工（勿混）

| 文件                    | 记什么                                                                                            | 典型例子                                                                                   |
| ----------------------- | ------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------ |
| **`findings.md`**       | 查码/审计**发现的问题**；P1-GATE **阻塞项**；**计划阶段** disposition（仍开放 / 已关闭 / 按设计） | F-07 fred 双轨 DISABLED；F-17 默认禁用按设计                                               |
| **`task_plan.md`**      | 切片 AC、计划内 AD、Quiz 已决、依赖与权威绑定                                                     | AD-11 macro production-live；Slice 2a-R4 步骤 6                                            |
| **`note.md`（本文件）** | **执行时**才出现、计划与权威**均未预见**的取舍与实现偏离                                          | 实现中发现须改 helper 签名；测试夹具需额外 env；临时与 design 不一致的 ponytail 及升级路径 |

**禁止：**

- 把 findings 里已登记的阻塞项「再记一遍」到 note（应关 findings disposition）
- 把应走 ADR/用户审阅的 **design 变更**只写 note
- 口头关账不写 note 也不写 **N/A**

## 如何使用

每开放切片勾 PASS 前：

1. 完成 **AC-CLOSE-1**（回读本切片 + 权威清单，见 `task_plan.md` 该切片末尾）。
2. 若本切片有**计划外**事项 → 在下方 **记账正文** 新增一节（用模板）。
3. 若**无计划外** → 在 `progress.md` 该切片行写 `AC-CLOSE-2: N/A`，**不必**在本文件占行。

### 单条记账模板（复制填空）

```markdown
### [Slice ID] · [简短标题] · YYYY-MM-DD

| 项             | 内容                                                  |
| -------------- | ----------------------------------------------------- |
| **暴露的问题** | （用户/运营体感；与哪条权威或计划预期不符）           |
| **计划外点**   | （为何计划/权威未覆盖）                               |
| **决策**       | （选了什么 / 未选什么；产品约束）                     |
| **实现**       | （路径、契约、配置、接缝；文件级要点）                |
| **验证**       | （pytest / CLI / 人工核对命令与结论）                 |
| **后续**       | （是否需回写 `findings.md` / 其他切片；无则写「无」） |
```
